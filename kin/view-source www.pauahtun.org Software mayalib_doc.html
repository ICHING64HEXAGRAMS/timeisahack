<font size=7>Mayalib &mdash; Mayan dates and numbers for Python</font>
<p>
<h2>Introduction</h2>
<blockquote>
Mayalib contains a full complement of classes, functions and a few variables suitable for
performing basic arithmetic operations with Mayan dates and numbers; some
support for astronomical functions is provided, but much in this area
is slated for future implementation. Familiarity with the Mayan calendar
is a prerequisite; see the <b>References</b> for resources that will help
you learn about that calendar. Familiarity with Python is not a prerequisite,
as this documentation offers a short section on basic programming.
Again, see <b>References</b> for more help. Many functions
and class methods pertain to the production of HTML, suitable for use
in web cgi programs. These last are covered in a separate section
of this document.

<p>
Mayalib and the contained classes, functions and variables were created in
order to make research into the form, function and structure of the Mayan
calendar as simple as possible. Python meshes with the Mayan calendar so
well it is hard to believe that the language was not created with the calendar
in mind, but that appears to be the case. The core implementation of the
language has arbitrarily long integers built in, so very little needs to be
done in order to support multi-trillion year calculations, commonplace in
work with the Mayan calendar. For further information, see the paper
I presented at the Seventh International Python Conference,
<a href="python_vuh.html"><cite>Python Vuh</cite>: Mayan Calendrical Mathematics with Python</a>.
</blockquote>
<h2>Variables in mayalib</h2>
<blockquote>
While there are a number of variables in the mayalib namespace,
most of their names include a leading underscore, so as not to
be visible outside the mayalib module.
The remaining
names were chosen for minimal collision with user variables; the preferred
method for using mayalib, contrary to recommended usage for most
other supplementary Python packages and libraries, is to include the
line
<p>
<center>
<pre>
from mayalib import *
</pre>
</center>
<p>
at the beginning of your programs. Naturally, you may still prefer to
simply <code>import mayalib</code> instead, but then you will have a lot
more typing to do.
<p>
Here are the names of exposed variables in mayalib and their meanings.
<p>
<dl>
<dt><b>MAYALIB_VERSION</b>: 1.1
<dd>The version number for the current mayalib
<dt><b>center, left, right</b>: 0, 1, 2
<dd>Used in building HTML tables; specifies the alignment in table elements
<dt><b>LOCATION</b>: REMOTE
<dd>Specifies a prefix for finding glyph images;
Can be one of LOCAL, RELATIVE or REMOTE.
REMOTE and RELATIVE should not be changed,
but LOCAL should be edited and changed to reflect
DocumentRoot on your webserver, if you have one
<dt><b>baktun, katun, tun, uinal, kin</b>: 144,000, 7,200, 360, 20, 1
<dd>Standard values for time periods
</dl>
</blockquote>

<h2>Utility Functions</h2>
<blockquote>
<dl>
<dt><b>isnumber(<i>s</i>)</b>
<dd>Argument: string<br>
Return value: 1 if <i>s</i> contains only digits 0-9, 0 otherwise
<dt><b>issignednumber(<i>s</i>)</b>
<dd>Argument: string<br>
Return value: 1 if <i>s</i> contains only digits 0-9 and signs, 0 otherwise
<dt><b>hasvalue(<i>d, m</i>)</b>
<dd>Arguments: dictionary, any<br>
Return value: returns the key for value <i>m</i> if it can be found in dictionary <i>d</i>,
otherwise <i>None</i>
<dt><b>exgcd(<i>u, v, pr = 0</i>)</b>
<dd>Arguments: <i>u, v</i> are integers<br>
Return value: calculates the <i>greatest common divisor</i> of <i>u</i> and <i>v</i>
using an algoritm called &lsquo;The Pulverizer&rsquo; as described in Knuth&rsquo;s <cite>The Art of
Computer Programming</cite>, Volume 2, Section 4.5.2. If <i>pr</i> is not 0,
prints the steps it takes to calculate the <i>gcd</i>. Returns a tuple
(gcd, mn1, mn2), where mn1 and mn2 are magic numbers useful in other calculations.
</dl>
</blockquote>
<h2>Class Mayanum</h2>
<blockquote>
Class <i>mayanum</i> provides full-function Mayan dates and Mayan numbers, and has
built in facilities for addition, subtraction, multiplication and other useful
mathematical operations. One of the key features is the ability to treat a mayanum
as either a calendar number <i>or</i> an ordinary (uniform base 20) number.
Creating a mayanum is a simple process. The simplest is
<p align=center>
<code>x = mayanum()</code>
<p>
which creates a mayanum for day 0 of the Mayan calendar, or <b>0.0.0.0.0</b>. Since
mayanums have no idea whether they will be used as dates or as ordinary numbers, creating
a mayanum builds only the Long Count coeffecient component. The Calendar Round and other
cycles must be separately added, by using a method called <i>calculate()</i>:
<p align=center>
<code>x = mayanum()<br>
x.calculate()</code>
<p>
Applying this method, as shown above, calculates the correct Calendar Round (<b>4 &rsquo;Ahaw 8 Kumk&rsquo;u</b>),
the correct Lord of the Night (<b>G9</b>), the 819-day count position (<b>3</b>), and various
other cycles. Creating mayanums for other dates is almost as simple; you merely need to provide
Long Count coeffecients for the date you wish:
<p align=center>
<code>x = mayanum([12, 19, 5, 14, 8])<br>
x.calculate()</code>
<p>
which manufactures a Mayan date for <b>12.19.5.14.8 1 Lamat 1 K&rsquo;ank&rsquo;in</b>,
or December 25 1998 (using the standard <b>584285</b> correlation; a means to change
the correlation date is described in the next section, <b>Non-Class Mayan Functions</b>).
<p>
Mathematics with mayanums can be very easy. They can be added, subtracted, multiplied (with care),
changed to a different style (date style to number style and back again), converted
to the <i>Gregorian</i> calendar and compared. As an example, here&rsquo;s how to add and multiply:
<p align=center>
<code>x = mayanum()<br>
x.calculate()<br>
z = (x + &ldquo;1.0.0.0.0&rdquo;) * 13 # Add one bak&rsquo;tun and multiply by 13<br>
z.calculate ( )<br>
print x, z<br>
print x.gregorian()<br>
print z.gregorian()</code>
<p>
The above steps will print the starting day and the day 13 bak&rsquo;tuns later when some
consider that the Mayan calendar will end. Many other operations are documented below.
Note that methods in the style &ldquo;<b>__repr__</b>&rdquo; (these are called &ldquo;special class
methods&rdquo;) are not usually called that way,
but by performing some other operation. For example, the <b>__repr__</b> is called
by typing <code>print x</code>, and the <b>__add__</b> method is called whenever you
type <code>x = x + y</code> when <code>x</code> and <code>y</code> are both
mayanums.
<p>
<dl>
<dt><b>mayanum(<i>lc=[0,0,0,0,0], t=-1, v=-1, hd=-1, hm=-1, g=-1</i>)</b>
<dd>This is the constructor for class mayanum. All arguments are optional.
The arguments, if used, are described below:
<p>
<dl>
<dt><b>lc</b>:
<dd>A Long Count <i>list</i>, which may be <i>any</i> length (<i>i.e.</i>,
the date from Coba causes no problems whatsoever). Refer to the Python
documentation for the definition of <i>list</i>.
<dt><b>t</b>:
<dd>The <i>trecena</i>, an integer in the range <b>1-13</b>.
<dt><b>v</b>:
<dd>The <i>veintena</i>, an integer in the range <b>0-19</b>,
representing the named days <b>&rsquo;Ahaw</b> to <b>Kawak</b>.
<dt><b>hd</b>:
<dd>The <i>haab day</i>, an integer in the range <b>0-19</b>.
<dt><b>hm</b>:
<dd>The <i>haab month</i>, an integer in the range <b>0-18</b>,
representing the named months <b>Pohp</b> to <b>Wayeb</b>.
<dt><b>g</b>:
<dd>The <i>G glyph</i>, an integer in the range <b>0-8</b>,
representing the Lord of the Night; 0 is G9, 8 is G8.
</dl>
<p>
Most of the time, it is easiest to simply create a mayanum with the default
arguments and later add or subtract to arrive at the particular date you&rsquo;re
interested in. All the methods built into mayanum are described below.
Note, however, that undocumented methods are undocumented for a reason, and
should not be used.
<p>
<dt><b>alautun()</b>
<dd>Arguments: none<br>
Return value: the <i>alautun</i> Long Count coeffecient <b>lc[8]</b>.
<dt><b>kinchiltun()</b>
<dd>Arguments: none<br>
Return value: the <i>kinchiltun</i> Long Count coeffecient <b>lc[7]</b>.
<dt><b>calabtun()</b>
<dd>Arguments: none<br>
Return value: the <i>calabtun</i> Long Count coeffecient <b>lc[6]</b>.
<dt><b>pictun()</b>
<dd>Arguments: none<br>
Return value: the <i>pictun</i> Long Count coeffecient <b>lc[5]</b>.
<dt><b>baktun()</b>
<dd>Arguments: none<br>
Return value: the <i>baktun</i> Long Count coeffecient <b>lc[4]</b>.
<dt><b>katun()</b>
<dd>Arguments: none<br>
Return value: the <i>katun</i> Long Count coeffecient <b>lc[3]</b>.
<dt><b>tun()</b>
<dd>Arguments: none<br>
Return value: the <i>tun</i> Long Count coeffecient <b>lc[2]</b>.
<dt><b>uinal()</b>
<dd>Arguments: none<br>
Return value: the <i>uinal</i> Long Count coeffecient <b>lc[1]</b>.
<dt><b>kin()</b>
<dd>Arguments: none<br>
Return value: the <i>kin</i> Long Count coeffecient <b>lc[0]</b>.
<p>
For the above coeffecient operations, if a coeffecient does not exist
the number is enlarged and the correct place is initialized with 0. For
numbers that contain places higher than <i>alautun</i>, the coeffecient
may easily be retrieved by simply indexing the number. <i>I.e.</i>,
the next place above <i>alautun</i> is obtained with
<p align=center>
<code>x = mayanum()<br>
x = x + &ldquo;13.13.13.13.13.13.13.13.13.13.13.13.0.0.0.0&rdquo;<br>
print x[9]</code>
<p>
However, when indexing a number, the indexed place must already exist;
see the methods <i>grow()</i> and <i>shrink()</i>, described later.
<p>
<dt><b>__repr__()</b>
<dd>Arguments: none<br>
This is the method called whenever a mayanum is printed or turned into
a string. It knows whether a date has been calculated or not, and adjusts
the conversion process accordingly; observe the difference between the results
of the following two <i>print</i> statements:
<p align=center>
<code>x = mayanum()<br>
print x<br>
x.calculate()<br>
s = &lsquo;x&lsquo;<br>
print s</code>
<p>
<dt><b>f819()</b>
<dd>Arguments: none<br>
Return value: calculates the 819-day count position using the Long Count coeffecients,
inserts the result into the mayanum and returns the calculated position, an integer
ranging from 0 to 818.
<dt><b>get819list()</b>
<dd>Arguments: none<br>
Return value: calculates the Calendar Round of the previous 819-day station and returns
it as a string.
<p align=center>
<code>x = mayanum()<br>
x.calculate()<br>
print x<br>
s = x.get819list()<br>
print s</code>
<p>
<dt><b>list_color_direction_text()</b>
<dd>Arguments: none<br>

['<img alt="Kan, yellow" title="Kan, yellow" src="/Gif/Kan_1c.png" >', ' ', '<img alt="South, Nohol" title="South, Nohol" src="/Gif/nohol_1.png" >']

<dd>Arguments: none<br>
Return value: none. Prints the text for the color and direction:

</center>

<dt><b>__getitem__(<i>key</i>)</b>
<dd>Arguments: integer <i>key</i><br>
This is the special class method that allows you to retrieve Long Count coeffecients
by their indices; for a <i>date</i> <b>12.19.5.14.8</b>, <i>date</i>[0] is 8, and <i>date</i>[4] is 12.
<dt><b>__getitem__(<i>key, value</i>)</b>
<dd>Arguments: integer <i>key, value</i><br>
Special class method that lets you set Long Count coeffecients by their indices; if <i>date</i>
is <b>12.19.5.14.8</b>, then <code><i>date</i>[4] = 13</code> results in the date being
changed to <b>13.19.5.14.8</b>.
<dt><b>__cmp__(<i>other</i>)</b>
<dd>Arguments: mayanum <i>other</i><br>
Special class method allowing comparisons, such that if three dates x, y and z
are <b>-6.13.0</b>, <b>0.0.0.0.0</b> and <b>6.13.0</b> respectively, then the
relation <i>x &lt; y $lt; z</i> holds.
<dt><b>__neg__()</b>
<dd>Arguments: none<br>
Special class method; changes the sign on the number. For example
if <i>x</i> is <b>12.19.5.14.8 1 Lamat 1 K&rsquo;ank&rsquo;in</b>,
then <i>-x</i> is <b>-12.19.5.14.8 7 &rsquo;Eb 10 Sots</b>. Performing the operation
again on the result, <i>-(-x)</i>, recovers the original number, <i>x</i>.
<dt><b>__pos__()</b>
<dd>Arguments: none<br>
Special class method; this is not a sign-changing operation, but the <i>identity</i>
operation. <i>I.e.</i>, <i>+x</i> gives you <i>x</i>, and <i>+(-x)</i> gives you
<i>-x</i>.
<dt><b>__abs__()</b>
<dd>Arguments: none<br>
This special class method is called whenever <i>abs()</i> is called with a mayanum
as the argument. Such values that result technically have no sign, but
<i>abs(-x)</i> and <i>-(-x)</i> have the same effect.
<dt><b>__coerce__(<i>other</i>)</b>
<dd>Arguments: <i>other</i><br>
This method is called whenever a mayanum is included within an expression with some other
type, such as <code>x + 13</code> or <code>x - &ldquo;13.0.0.0.0&rdquo;</code>. The types that are
useful in such expressions include:
<ul>
<li><i>mayanum</i>
<li><i>integer</i>
<li><i>long</i>
<li><i>string</i>
<li><i>list</i>, although only lists such as <b>[1,0,0,0,0]</b>, composed entirely of numbers,
can be dealt with.
</ul>
<dt><b>__sub__(<i>minus</i>)</b>
<dd>Arguments: mayanum <i>minus</i><br>
Special class method that is called (after the <b>__coerce__</b> method, if required) whenever
a mayanum is used in an expression involving subtraction and the mayanum is on the left of such
an expression. Performs accurate signed arithmetic. If x is <b>-0.13</b>, then <code>x = x - 13</code>
will result in <b>-0.1.6</b>.
<dt><b>__rsub__(<i>minus</i>)</b>
<dd>Arguments: mayanum <i>minus</i><br>
Same as <b>__sub__()</b>, but called only when the mayanum is on the right side of
an expression. If x is <b>-0.13</b>, then <code>x = 13 - x</code> will give
<b>0.1.6</b>.
<dt><b>__add__(<i>plus</i>)</b>
<dd>Arguments: mayanum <i>plus</i><br>
Special class method that is called (after the <b>__coerce__</b> method, if required) whenever
a mayanum is used in an expression involving addition and the mayanum is on the left of such
an expression. Performs accurate signed arithmetic. If x is <b>-0.13</b>, then <code>x = x + 13</code>
will result in <b>0.0</b>.
<dt><b>__radd__(<i>plus</i>)</b>
<dd>Arguments: mayanum <i>plus</i><br>
Same as <b>__sub__()</b>, but called only when the mayanum is on the right side of
an expression. If x is <b>-0.13</b>, then <code>x = -13 + x</code> will give
<b>-0.1.6</b>.
<dt><b>__mul__(<i>multiplier</i>)</b>
<dd>Arguments: mayanum <i>multiplier</i><br>
Multiplication of a mayanum can only be done by single integers, not by a
full-fledged mayanum. This is because multiplication of two mixed-radix numbers
cannot be performed&mdash;think of multiplying pounds, shillings and pence times another
amount in pounds, shillings and pence; it doesn&rsquo;t make sense.
Thus, <b>2.5.19</b> &middot; 20 works, but <b>2.5.19</b> &middot; <b>1.0</b> doesn&rsquo;t.
Signed multiplication by one small integer is correctly supported, however.
<dt><b>__lshift__(<i>n</i>)</b>
<dd>Arguments: int <i>n</i><br>
This method is called for the shift operator, <b>&lt;&lt;</b>.
Shifts <i>mayanum</i> left by <i>n</i> places, re-evaluating the value of each place
if necessary. For a pure radix-20 <i>mayanum</i>, only the shift is performed. For
a normal, date, <i>mayanum</i>, re-evaluation is necessary. For example, if <i>mayanum</i>
is <b>0.19</b>, then the result of <code><i>mayanum</i> &lt;&lt; 1</code> is <b>1.1.0</b>,
and the result of <code><i>mayanum</i> &lt;&lt; 2</code> is <b>19.0.0</b>. The reasons for
this behaviour are left for the reader to discover.
<dt><b>mulmaya(<i>mul</i>)</b>
<dd>Arguments: mayanum <i>mul</i><br>
This is a standard method, which converts both <i>mayanum</i>s to Python longs before
performing multiplication. The resulting Python long is converted to a <i>mayanum</i>
and returned. For example, if <i>x</i> is <b>19.0.0</b> and <i>mul</i> is <b>2.4.19</b>,
then the result of <code>x.mulmaya(mul)</code> is <b>1.18.18.1.0.0</b>. The difference
between this and the special class method <b>__mul__</b> is that <i>mulmaya</i> <i>always</i>
converts its operands to Python longs, whereas <b>__mul__</b> only converts the multiplier
to an integer. <i>mulmaya</i> can also only be used when both operands are <i>mayanum</i>s.
<dt><b>divmaya(<i>div</i>)</b>
<dd>Arguments: mayanum <i>div</i><br>
This is exactly similar to <i>mulmaya</i>, in that both operands are converted to Python longs
from <i>mayanum</i>s, and the result of the division is turned into a <i>mayanum</i>.
<dt><b>modmaya(<i>mod</i>)</b>
<dd>Arguments: mayanum <i>mod</i><br>
This method is exactly like <i>divmaya</i> except that it returns the remainder from
the division as a <i>mayanum</i>.
<dt><b>divmodmaya(<i>div</i>)</b>
<dd>Arguments: mayanum <i>div</i><br>
This method is a combination of <i>divmaya</i> and <i>modmaya</i>, and returns both the quotient
and the remainder (in that order) as a tuple.
<dt><b>B20()</b>
<dd>Arguments: none<br>
Converts <i>mayanum</i> to a pure base 20 Mayan number; if the number is already
base 20, it does nothing.
<dt><b>B18()</b>
<dd>Arguments: none<br>
Converts <i>mayanum</i> to a normal base 18 Mayan date; if the number is already
base 18, it does nothing.
<dt><b>b20()</b>
<dd>Arguments: none<br>
Converts <i>mayanum</i> to a pure base 20 Mayan number by converting to a Python
long and then back to a <i>mayanum</i>. Deprecated.
<dt><b>b18()</b>
<dd>Arguments: none<br>
Converts <i>mayanum</i> to a normal base 18 Mayan date by converting to a Python
long and then back to a <i>mayanum</i>. Deprecated.
<dt><b>submaya(<i>minus</i>)</b>
<dd>Arguments: mayanum <i>minus</i><br>
Performs a subtraction of <i>minus</i> from <i>mayanum</i> by first converting
both operands to Python longs. Deprecated.
<dt><b>__long__()</b>
<dd>Arguments: none<br>
This method is called when it is required to convert a <i>mayanum</i> to
a Python long, as in <code>x = long(<i>mayanum</i>)</code>.
<dt><b>__int__()</b>
<dd>Arguments: none<br>
This method is called when it is required to convert a <i>mayanum</i> to
a Python int, as in <code>x = int(<i>mayanum</i>)</code>.
<dt><b>__float__()</b>
<dd>Arguments: none<br>
This method is called when it is required to convert a <i>mayanum</i> to
a Python float, as in <code>x = float(<i>mayanum</i>)</code>. However,
be warned that this returns the float equivalent of the Julian Period
day number, so that the result depends upon the value of the Correlation
constant.
<dt><b>calculated()</b>
<dd>Arguments: none<br>
Returns 1 if fields other than the Long Count coeffecients have been
calculated. If 1 is returned, all components of the class have been
calculated, and if 0 is returned, only the Long Count coeffecients
can be relied upon.
<dt><b>longtomaya(<i>n</i>)</b>
<dd>Arguments: long <i>n</i><br>

<dt><b>mayatolong()</b>

<dt><b>reset()</b>

<dt><b>__len__()</b>

<dt><b>maxdays(<i>n</i>)</b>
<dd>Arguments: integer <i>or</i> list <i>n</i><br>
Returns the maximum possible number of days for a Long Count.
If <i>lc</i> is a list (a Long Count), then <i>maxdays</i> will return

If <i>n</i> is an integer, it returns the maximum possible number of days for a Long Count of

will return 2,880,000.
<dt><b>lineardistributionofn(<i>ind, n, cycle = 365, spl = 20, sd = 348</i>)</b>
<dd>Arguments: integer <i>ind</i>, <i>n</i>, <i>cycle</i>, <i>spl</i>, <i>sd</i><br>
Return value: number of days of hunk <i>ind</i> contained within a period of time <i>n</i>
(in days) where the hunk size is <i>spl</i> and the starting day of the cycle is <i>sd</i>.
This method was originally set up to calculate how many days of <i>haab month</i> <i>ind</i>
were contained within <i>n</i> days of the Long Count starting at day 0, but I soon realized
that it could be generalized. As it is, if you want to determine how many days of month
<b>Kumk&rsquo;u</b> are contained in the first four katuns since day 0, the syntax would be<br>
<code>t = x.lineardistributionofn(17, 28800, 365, 20, 348)</code><br>
and the answer is 1572.
<dt><b>distributionofn(<i>day, n, cycle = 13, sd = 4</i>)</b>
<dd>Arguments: integer <i>day</i>, <i>n</i>, <i>cycle</i>, <i>spl</i>, <i>sd</i><br>
Similar to <i>lineardistributionofn()</i>, except that it works only for modular
cycles such as the <i>trecena</i> or the <i>Calendar Round</i>. For example,
if you wish to know how many days <b>&rsquo;Ahaw</b> there were in the first four katuns,
then enter <code>x.distributionofn(0,28800,13,4)</code> into the interpreter (assuming
that x is a <i>mayanum</i>). The answer is 2215.
